# GnG Монополия 🎲

Веб-приложение на FastAPI, реализующее настольную игру "Монополия" с уникальной механикой - каждое поле на доске соответствует случайной игре.

## 🎯 Концепция игры

1. **Регистрация** - игроки создают аккаунты и игровые профили
2. **Сессии** - создание или присоединение к игровым сессиям
3. **Ходы** - бросок кубиков и перемещение по полю
4. **Игры** - прохождение случайных игр с выбранными условиями
5. **Предметы** - использование предметов для модификации игрового процесса
6. **Статистика** - отслеживание прогресса и победных очков

## 🏗️ Архитектура

```
Frontend (React/Vue) ←→ FastAPI Backend ←→ PostgreSQL + Redis
```

### Основные компоненты:
- **User** - аккаунт пользователя
- **Player** - игровой персонаж с данными
- **Session** - игровые сессии
- **Cell** - клетки игрового поля
- **Game** - выпавшие игры
- **Item/ItemInstance** - система предметов

## 🚀 Быстрый старт

### Требования
- Docker и Docker Compose
- Python 3.11+

### Запуск

1. **Клонирование репозитория:**
```bash
git clone <repository-url>
cd GnG-Monopoly
```

2. **Настройка переменных окружения:**
```bash
cp backend/docker.env.example backend/docker.env
# Отредактируйте docker.env с вашими настройками
```

3. **Запуск через Docker:**
```bash
docker-compose up --build
```

4. **Применение миграций:**
```bash
docker exec -it backend alembic upgrade head
```

5. **Доступ к приложению:**
- Backend API: http://localhost:5000
- API Documentation: http://localhost:5000/docs

## 🎮 API Endpoints

### Основные эндпоинты:
- `POST /api/v1/register` - Регистрация пользователя
- `POST /api/v1/sessions` - Создание сессии
- `POST /api/v1/sessions/{id}/join` - Присоединение к сессии
- `POST /api/v1/game/turn` - Выполнение хода
- `GET /api/v1/players/{id}/dashboard` - Дашборд игрока

## 🔧 Разработка

### Локальная разработка:
```bash
# Установка зависимостей
cd backend
pip install -r requirements.txt

# Запуск PostgreSQL
docker-compose up db

# Запуск приложения
uvicorn src.start:app --reload --port 5000
```

### Тестирование:
```bash
cd backend
pytest tests/
```

## 🎯 Игровая механика

### Ход игрока:
1. Бросок 2 кубиков (d6)
2. Применение модификаторов
3. Перемещение по полю
4. Выбор действия на клетке
5. Прохождение игры или выполнение условия

### Типы клеток:
- **Старт** - начальная позиция
- **Игровая** - случайная игра
- **Тюрьма** - наказание за дроп
- **Магазин** - покупка предметов
- **Особая** - специальные события

### Статусы игр:
- **PROGRESS** - в процессе
- **COMPLETED** - завершена
- **DROP** - дропнута
- **REROLLED** - переброшена
